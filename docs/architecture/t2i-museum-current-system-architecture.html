t2i_museum Current System Architecture

Chrome extension driven ingestion for Jimeng work-detail pages, backed by a local-first collector, SQLite catalog, local image cache, and online style normalization for the evolving prompt-style knowledge base.

Ingestion Loop

  • • Scope is intentionally limited to Jimeng image detail pages.
  • • The extension stays thin: inject UI, scrape DOM, send raw payload.
  • • Collector owns durability, dedupe, retries, and analysis orchestration.

Knowledge Model

  • • Raw work facts are stored first, derived style knowledge is layered on top.
  • • Canonical styles, aliases, and work-style links allow vocabulary growth over time.
  • • Online LLM assists discovery, but local normalization logic makes the final mapping decision.

MVP Constraints

  • • Assumes your local Chrome is already logged into Jimeng.
  • • Images are cached immediately because signed source URLs expire.
  • • Museum is read-only for now: browse works, inspect prompts, and jump into style pages.
